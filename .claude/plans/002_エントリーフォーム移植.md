# 002: エントリーフォーム移植

## 概要
`hackit-for-participant` のイベント申込フォームを `Hackit2026` に移植し、実運用に耐えうる品質にする。

## 方針
- UI/フロントは基本そのままコピー（EventForm.tsx）
- カラースキームを Hackit2026 のテーマ（オレンジ系 #FF6B35）に合わせる
- バックエンド: GAS + Google Sheets を維持（スプレッドシートにまとまる要件）
- スコープ: イベント申込のみ（チーム登録・成果物提出は対象外）
- 1ブランチ（`feature/entry-form`）で全作業 → ローカル確認後にPR1本をmainへ

## ブランチ戦略
- `feature/entry-form` で全作業
- コミットは細かく分割（50行/コミット目安）
- ローカルで動作確認後、PR1本をmainに出す

## 実装内容

### エージェント1: フォームUI移植
- `/entry` ページ作成（`src/app/entry/page.tsx`）
- EventForm コンポーネント移植（`src/app/entry/components/EventForm.tsx`）
  - カラースキーム: amber系 → Hackit2026テーマに変換
- Countdown コンポーネント移植（`src/app/entry/components/Countdown.tsx`）
- DeadlineBanner コンポーネント移植（`src/app/entry/components/DeadlineBanner.tsx`）
- useSettings フック移植（`src/app/entry/hooks/useSettings.tsx`）
- CTAボタンのリンクを `/entry` に変更

### エージェント2: APIルート・バックエンド + 本番品質
- `/api/submit` ルート作成（`src/app/api/submit/route.ts`）
- `/api/settings` ルート作成（`src/app/api/settings/route.ts`）
- GAS コード（`gas/Code.gs`）— リポジトリに含めて管理
- セキュリティミドルウェア追加
- 二重送信防止（フロント: isSubmitting / サーバー: 冪等性キー）
- サーバーサイドバリデーション
- レート制限
- Origin チェック

## カラーマッピング
| 用途 | 元 (amber系) | 変換先 (Hackit2026) |
|------|-------------|-------------------|
| メインアクセント | amber-500/600 | #FF6B35 / #FF8C5A |
| ボタンhover | amber-600 | #e55e2b |
| ボーダー | amber-300 | #FFB380 |
| リング/フォーカス | amber-200 | #FFD4B0 |
| 背景薄い | amber-50/100 | #FFF5EE |
| テキスト | amber-900 | base-dark (#2d2d2d) |
| バッジ背景 | amber-100 | #FFE8D6 |

## 環境変数（Vercelに設定が必要）
```
GAS_WEBAPP_URL=<GASデプロイURL>
DEV_MODE=1（開発時）/ 0（本番）
```

## Google Sheets 準備（ユーザー作業）
1. スプレッドシート作成（Events, Personal, Settings の3シート）
2. GAS コードをデプロイ
3. Vercel に環境変数を設定

## リスク
- EventForm.tsx が14,600行と巨大 → 新規ファイル追加なので200行制限の例外
- GAS/Sheets の設定はユーザー側作業が必要
